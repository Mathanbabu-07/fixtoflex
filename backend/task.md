Development Prompt — Recruiter Gmail Integration (Phase 1)
Objective

Implement Gmail OAuth integration for the Recruiter UI by reusing the existing Google OAuth and Gmail Draft infrastructure. The recruiter authenticates with their own Google account, grants Gmail permissions, and can create personalized Gmail drafts for matched candidates directly from the recruiter dashboard.

1. Reuse Existing Google OAuth

Do not create a second OAuth implementation.

Reuse the existing:

OAuth Client ID
OAuth Client Secret
Gmail API service
Token refresh logic
Gmail draft creation service

Only create recruiter-specific routes and session handling.

Maintain complete separation between Candidate and Recruiter sessions.

2. Recruiter Gmail Login

Add a "Connect Gmail" button in the Recruiter header.

3. Successful Login

After successful authorization:

Hide the Connect Gmail button.
Display:
Gmail profile photo
Recruiter name
Gmail address
Show a success toast:
✓ Gmail connected successfully.
Your account is ready to create recruiter email drafts.

Restore this state automatically using the stored refresh token.

10. Error Handling

Gracefully handle:

Recruiter not connected to Gmail
Expired OAuth token (refresh automatically)
Gmail API failure
Missing candidate email
Network errors

Display clear user-friendly messages without exposing technical details.

11. Security
Reuse the existing OAuth implementation.
Store recruiter OAuth tokens separately from candidate tokens.
Never reuse candidate Gmail credentials.
Always create drafts in the currently authenticated recruiter's Gmail account.
Refresh access tokens automatically when expired.
12. Out of Scope (Do Not Implement)
Automatic email sending
Bulk draft creation
Send All
Attach resume or files
CC/BCC
Reply tracking
Email templates library
